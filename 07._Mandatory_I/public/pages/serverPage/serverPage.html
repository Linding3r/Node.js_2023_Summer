Anatomy of an URL

            
https://www.google.com/search?client=firefox-b-d&q=test

protocol    subdomain    domain    tld(top-level domain)    endpoint       query String      
https://    www.         google           .com              /search     ?client=firefox-b-d&=test
  • Protocol: It specifies how the web browser should communicate with the web server. In this case, it's "https://" which stands for Hypertext Transfer Protocol Secure, indicating a secure, encrypted connection.

  • Subdomain: "www" is a commonly used subdomain and typically represents the World Wide Web.

  • Domain: "google" is the main domain name, indicating the specific website you're trying to access.

  • TLD (Top-Level Domain): ".com" is a TLD, which often signifies that it's a commercial website.

  • Endpoint: "/search" is the path or location on the web server where the specific resource or page is located.

  • Query String: This part starts with "?" and includes parameters like "client=firefox-b-d" and "=test." These parameters are often used to pass information to the web server for various purposes, such as performing a search ("test" in this case) or customizing the user experience based on the client (Firefox in this case).
