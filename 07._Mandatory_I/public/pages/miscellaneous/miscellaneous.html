Anatomy of an URL

            
https://www.google.com/search?client=firefox-b-d&q=test

protocol    subdomain    domain    tld(top-level domain)    endpoint       query String      
https://    www.         google           .com              /search     ?client=firefox-b-d&=test
  • Protocol: It specifies how the web browser should communicate with the web server. In this case, it's "https://" which stands for Hypertext Transfer Protocol Secure, indicating a secure, encrypted connection.

  • Subdomain: "www" is a commonly used subdomain and typically represents the World Wide Web.

  • Domain: "google" is the main domain name, indicating the specific website you're trying to access.

  • TLD (Top-Level Domain): ".com" is a TLD, which often signifies that it's a commercial website.

  • Endpoint: "/search" is the path or location on the web server where the specific resource or page is located.

  • Query String: This part starts with "?" and includes parameters like "client=firefox-b-d" and "=test." These parameters are often used to pass information to the web server for various purposes, such as performing a search ("test" in this case) or customizing the user experience based on the client (Firefox in this case).

Git 101

Git is a version control system that tracks changes in your files, allowing you to collaborate with others, keep a history of your work, and manage different project versions through branches. It's essential for software development and any text-based file management. Here is a list of some of the main git commands used:

Initialize a new Git repository in the current directory.

                
git init

Create a copy of a remote repository on your local machine.

                
git clone <repository_url>

Stage changes for commit.

                
git add <file_name>

Commit your staged changes with a descriptive message.

                
git commit -m "Your commit message"

Fetch and merge changes from a remote repository into your local branch.

                
git pull

Push your local commits to a remote repository.

                
git push

List all branches in your repository.

                
git branch

Switch to a different branch or commit.

                
git checkout <branch_name or commit_hash>

Merge changes from one branch into the current branch.

                
git merge <branch_name>

REST API Conventions:

  • Use Nouns for Resource Names: Resource URIs should use nouns to represent resources, not verbs. For example, /users is a better URI than /getUsers.

  • Use Plural Nouns: Resource names should be plural nouns to indicate collections. For example, /users represents a collection of user resources.

  • Use HTTP Methods Appropriately:

    • GET: Retrieve data from a resource.
    • POST: Create a new resource.
    • PUT: Update an existing resource.
    • DELETE: Remove a resource.
    • Use other HTTP methods when appropriate (e.g., PATCH for partial updates).

  • Use Status Codes: Return appropriate HTTP status codes (e.g., 200 OK, 201 Created, 404 Not Found) to indicate the outcome of API requests. These codes provide valuable information to clients.
